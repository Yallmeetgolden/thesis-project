# Photo AI Culling Tool

## Goal

Build an AI-assisted photo culling system that helps photographers **rank and select the best photos** from large batches by analyzing sharpness, lighting, contrast, and redundancy — while keeping the final deletion decision in the user’s control.

---

## Core Use Case

* Input: 300–1000 photos from a shoot
* Output:

  * Ranked list of photos (best → worst)
  * Recommended **Top N (e.g. 100)** photos
  * Grouping of similar / repeated shots
  * Clear reasons: blurry, bad lighting, duplicate

---

## System Architecture (High Level)

Frontend (React)
→ Backend API (FastAPI)
→ Image Analysis Pipeline (Python)
→ Scoring & Ranking Engine
→ Results (JSON)

Frontend only **displays results**. All decisions happen in the backend.

## Phase 1 — Image Quality Scoring (FOUNDATION)

This phase answers: **“How good is each photo?”**

### Metrics Implemented

#### 1. Sharpness Scoring

* Method: Laplacian variance
* Purpose: Detect blur

#### 2. Exposure Scoring

* Method: Mean brightness distance from mid-gray
* Purpose: Penalize under/over-exposed images

#### 3. Contrast Scoring

* Method: Standard deviation of grayscale histogram
* Purpose: Detect flat images

#### 4. Final Quality Score

Weighted combination:

```
Final Score =
0.4 * Sharpness
+ 0.3 * Exposure
+ 0.3 * Contrast
```

### Tools

* Python
* OpenCV
* NumPy

### Output (per image)

```json
{
  "filename": "img_001.jpg",
  "sharpness": 0.82,
  "exposure": 0.74,
  "contrast": 0.65,
  "final_score": 0.76
}
```

### Deliverables

* Script that analyzes a folder of images
* JSON file with scores for all images

## Phase 3 — Duplicate & Repeated Pose Handling

This phase prevents near-identical photos from dominating the Top N.

### Steps

#### 1. Image Embeddings

* Use pretrained CNN to convert images to vectors

#### 2. Similarity Measurement

* Cosine similarity between embeddings

#### 3. Clustering

* Group similar photos

#### 4. Duplicate Penalty

* Keep best photo per cluster
* Penalize others in ranking

### Tools

* PyTorch
* Pretrained model (ResNet or CLIP)
* scikit-learn

### Output

```json
{
  "group_id": 1,
  "images": ["a.jpg", "b.jpg", "c.jpg"],
  "best": "b.jpg"
}
```
